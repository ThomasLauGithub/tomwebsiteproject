Now, a brief word about Trello. Trello is advantageous for its ease of use to get started on managing a project or managing workflow for a particular project, yet there are many alternatives for effective kanban tools which may be preferred over Trello. [15] For example, some users may like to manipulate in greater detail how they make use of the kanban system and modify the code for how their tool operates. There are several competing kanban tools to Trello which are classed as ‘open source’ which would allow for greater personalisation if someone wanted to see exactly how their tools work. Two such examples of these are Taiga [16] and Restya [17]. Github is useful for version control of code.

Outcome 3 (Also a Review)

3.1 Review the benefits and drawbacks of IT tools and systems used in terms of productivity and efficiency

One way I could have been more efficient when making my website would be to define SMART goals. These goals would be specific, measurable, achievable, realistic and time-based. This would have been especially effective for boosting productivity, considering the amount of time I (lost) spent reading and learning about CSS. Time seemed to vanish as I was often sidetracked into reading about additional CSS tools and techniques beyond the basics, intrigued by more and more possibilities, realising only with hindsight that many of them were not necessary for this project. Website creation and design could have been done it in teams! (See below)

3.2 Describing ways to improve productivity and efficiency

Working in a team could have been superior to working alone on my project if tasks were split between team members. For example, the functionality of linking pages to each other within a website could be assigned to one person, whilst the matter of designing the aesthetic appearance of buttons used for navigating these pages could be given to another team member. Teams can provide constructive critique to other teams about each other’s web pages, with a broader range of feedback likely to be received due to more people being involved on each project.

3.3 Developing solutions to improve my own productivity using IT in digital projects

As someone whose strength lies in being a team player, more naturally inclined to work with, discuss ideas with, and encourage peers, something I would do differently next time would be to form a team using Slack as a means of communication and marking progress.

3.4 How to go about testing digital solutions

Work can be presented on a GitHub repository where it is made public and can be critiqued by others. Seeking other forums where work can be shared and discussed would be effective for peer review. However, specifically speaking, testing digital solutions can be performed by dedicated software testing companies such as Qualitest. It is stated that ‘as well as functional, security, performance and load testing, digital testing should cover API, network, machine learning and big data testing.’ [18]
